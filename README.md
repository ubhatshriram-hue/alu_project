A parameterized Arithmetic Logic Unit (ALU) was designed using synthesizable Verilog HDL. The ALU supports configurable data width through the `WIDTH` parameter, with a default width of 8 bits, making the design scalable and reusable for different digital applications. The design operates synchronously on the rising edge of the clock (`CLK`) with an active-high reset (`RST`). The ALU supports dual operating modes, where `MODE = 1` enables arithmetic operations and `MODE = 0` enables logical operations. Arithmetic functions include addition, subtraction, increment, decrement, compare, signed arithmetic, and multiplication-based operations, while logical functions include AND, NAND, OR, NOR, XOR, XNOR, shift, rotate, and NOT operations.

The ALU architecture is fully parameterized, allowing easy scalability to 4-bit, 8-bit, 16-bit, and 32-bit implementations without modifying the main design. Most operations are completed within a single clock cycle, while multiplication-related operations require two clock cycles because of pipeline support. Additional status outputs such as `COUT`, `OFLOW`, `G`, `L`, and `E` are generated to indicate carry, overflow, and comparison results respectively. An `ERR` flag is implemented to detect invalid command combinations or unsupported input conditions.

A self-checking Verilog testbench was developed to verify the ALU functionality. The verification environment instantiates both the Design Under Test (DUT) and a golden reference model, where identical input stimuli are applied to both modules and all outputs are automatically compared during simulation. The testbench validates arithmetic operations, logical operations, overflow conditions, comparison results, reset functionality, and invalid command detection. Future improvements include enhancing overflow handling, extending support for multiple parameter values, adding constrained-random verification, integrating formal verification properties, and improving pipeline efficiency for high-speed applications.
